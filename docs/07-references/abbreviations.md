# Сокращения и аббревиатуры

## Архитектурные термины

- **ADR** - Architecture Decision Record (Запись архитектурного решения)
- **API** - Application Programming Interface
- **BFF** - Backend for Frontend
- **C4** - Context, Containers, Components, Code (модель архитектуры)
- **DDD** - Domain-Driven Design
- **ERD** - Entity Relationship Diagram
- **MVP** - Minimum Viable Product
- **NFR** - Non-Functional Requirements (Нефункциональные требования)

## Технологии

- **DB** - Database (База данных)
- **ERP** - Enterprise Resource Planning
- **Kafka** - Apache Kafka (платформа потоковой обработки данных)
- **REST** - Representational State Transfer
- **SLA** - Service Level Agreement
- **SLO** - Service Level Objective
- **SPA** - Single Page Application

## Требования

- **BR** - Business Requirements (Бизнес-требования)
- **MoSCoW** - Must, Should, Could, Won't (методология приоритизации)
- **NFR** - Non-Functional Requirements (Нефункциональные требования)
- **SR** - System Requirements (Системные требования)

## Домены

- **ACC** - Accounts (Аккаунты)
- **CAT** - Catalog (Каталог)
- **CHK** - Checkout (Оформление заказа)
- **CLI** - Client/Customer (Клиент)
- **CRT** - Cart (Корзина)
- **INV** - Inventory (Склад/Запасы)
- **MKT** - Marketing (Маркетинг)
- **ORD** - Order (Заказ)
- **SUP** - Support (Поддержка)
- **X-BUS** - Cross-Business (Сквозные бизнес-требования)

## Процессы

- **CI/CD** - Continuous Integration / Continuous Deployment
- **QA** - Quality Assurance
- **UAT** - User Acceptance Testing
