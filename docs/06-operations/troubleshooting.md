# Руководство по устранению неполадок

## Типичные проблемы

### Проблемы с производительностью

#### Медленный ответ API

**Симптомы**: Высокий latency, таймауты

**Диагностика**:
1. Проверить метрики CPU/Memory
2. Проверить медленные запросы к БД
3. Проверить логи на ошибки

**Решение**:
- Оптимизировать запросы к БД
- Добавить индексы
- Масштабировать сервис

#### Высокий Kafka lag

**Симптомы**: Задержка обработки событий

**Диагностика**:
1. Проверить метрики consumer lag
2. Проверить производительность consumer'ов
3. Проверить размер топиков

**Решение**:
- Увеличить количество партиций
- Масштабировать consumer'ы
- Оптимизировать обработку сообщений

### Проблемы с доступностью

#### Сервис недоступен

**Симптомы**: 503 ошибки, health check failed

**Диагностика**:
1. Проверить статус контейнеров
2. Проверить логи на ошибки
3. Проверить ресурсы (CPU, Memory)

**Решение**:
- Перезапустить сервис
- Проверить зависимости (БД, Kafka)
- Масштабировать ресурсы

### Проблемы с данными

#### Несоответствие данных

**Симптомы**: Неправильные остатки, статусы заказов

**Диагностика**:
1. Проверить логи событий Kafka
2. Проверить транзакции в БД
3. Проверить интеграции

**Решение**:
- Исправить данные вручную
- Пересинхронизировать через события
- Проверить обработку событий

## Процедуры восстановления

### Откат версии

1. Определить стабильную версию
2. Остановить текущую версию
3. Развернуть предыдущую версию
4. Проверить работоспособность

### Восстановление БД

1. Определить точку восстановления
2. Восстановить из backup
3. Применить миграции (если необходимо)
4. Проверить целостность данных
