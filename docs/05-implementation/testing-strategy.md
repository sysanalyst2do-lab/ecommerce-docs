# Стратегия тестирования

## Пирамида тестирования

```
        /\
       /  \
      / E2E \          Небольшое количество
     /--------\
    /          \
   / Integration \    Среднее количество
  /--------------\
 /                \
/   Unit Tests     \   Большое количество
\------------------/
```

## Типы тестов

### Unit тесты

- **Цель**: Проверка отдельных компонентов и функций
- **Покрытие**: Бизнес-логика, утилиты, сервисы
- **Инструменты**: pytest (backend), Jest (frontend)
- **Целевое покрытие**: 80%+

### Integration тесты

- **Цель**: Проверка взаимодействия компонентов
- **Область**: API endpoints, взаимодействие с БД, внешние интеграции
- **Инструменты**: pytest с testcontainers, Supertest

### E2E тесты

- **Цель**: Проверка полных пользовательских сценариев
- **Критические сценарии**:
  - Поиск и просмотр товара
  - Добавление в корзину и оформление заказа
  - Регистрация и авторизация
  - Отслеживание заказа

- **Инструменты**: Playwright, Cypress

## Тестовые данные

- Использовать фикстуры для unit тестов
- Testcontainers для integration тестов
- Изолированные тестовые БД

## CI/CD

- Автоматический запуск тестов при каждом коммите
- Блокировка merge при падении тестов
- Отчёты о покрытии кода

## Статус

Раздел в разработке. Детальная стратегия будет уточнена по мере развития проекта.
